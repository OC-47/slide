どこにいてもインターネットとつながることのできる現在、
ワイヤレスネットワークは多様化している。

この章では、

  • ワイヤレスネットワークのパフォーマンスの基礎

説明されている。

ワイヤレス通信における最大のチャネル容量

ワイヤレス通信方式には、最大のチャネル容量が存在する。

チャネル容量とは、
受信 → 送信とデータを送信する際に通る伝送路を介して
確実に伝送できる情報の量のこと

このチャネル容量は、基本的な原則原理によって決定する。

最大のチャネル容量の計算方法

最大のチャネル容量の計算方法がある。



シャノン・ハートレーの定理である

  • BW : 周波数帯域幅(Hz)
  • S/N : シグナル/ノイズ(ワット)

チャネル容量に影響する2要素

チャネル容量に影響する2つの要素が以下である。

  • 利用可能な帯域幅の大きさ
  • 受信・送信間の信号強度

である。

ワイヤレスネットワークには様々な種類があるけれど、
基礎的な制約事項は共通している。

帯域幅とは

  • 無線通信は電波・電磁波といった共有媒体を使用している

  • ワイヤレスネットワークの規格により、使用できる帯域は定義されている
    • 例 : 802.11b, 802.11gは、2.4-2.5GHzの周波数帯を使用する

  • 周波数帯の割り当ては、国や地域の政府が取り決めている
    • 国や地域によって、動作しなくなったりする

  • 定理から、他の条件が同等である場合、チャネル容量は周波数帯域に比例することがわかる

周波数帯域によるパフォーマンス

使用する周波数帯域によってパフォーマンスが変わる

  • 低周波数
    • 遠くに伝わる
    • マクロセル : 大きなエリアをカバーする
    • ラジオなどのブロードキャストなど向き
  • 高周波数
    • 多くのデータを転送できる
    • 遠くにはつたわらない
    • マイクロセル : カバレッジ小、多数設備必要
    • 双方向通信

信号強度について

  • 送信側、受信側間の信号強度
  • 信号対雑音比、S/N比、SNRともいう
  • 信号レベルとノイズや干渉の比の値である
    • バッググラウンドノイズが大きいほど、情報伝達信号が強くなければならない
    • バッググラウンドノイズとは他のデバイスによる干渉をいう

パフォーマンスの現状

理想の環境

  • バッググラウンドノイズなし
  • 特定の周波数帯域を独占使用

現状ほとんどありえない

  • 帯域幅の不足
  • ワイヤレスデバイス多すぎ

その他の要因 → 変調

帯域幅、SN比といった物理的要素に続き、
信号の符号化アルゴリズムも重大な影響をもっている

アナログからデジタルへの変換のことである。

この変換のことを、変調と呼ぶ。
変調は、変調アルファベットによって変換効率が変わる。
変調アルファベットとシンボルレートの組み合わせで
チャネルのスループットが決まる。

パフォーマンスを下げる原因

パフォーマンスに影響を与える要素として、

  • 送受信の距離
  • バックグラウンドの大きさ
  • 同じネットワーク内でのユーザによる干渉
  • 近くの異なるネットワークのユーザによる干渉
  • 送受信者の利用可能な出力
  • 計算能力と変調アルゴリズム

理想のパフォーマンス達成のためには

現状、望み通りのデータ転送速度の達成には・・・

  • ノイズや干渉を取り除く
  • 送受信の距離を縮める
  • 出力高め、信号強度を上げる
  • 最高の変調アルゴリズムを選択する
  • 有線ネットワークを利用する

6章

この章では、

  • WiFiのパフォーマンス

について説明されている。

WIfiとは

  • WiFiとは
    • 正式名称 : wIfI aLLIANCE
    • 一般的に、ieee 802.11標準規格に基づいたすべての製品

Ethernet → 無線LAN

  • ワイヤレス規格(802.11)は、Ethernet規格(802.3)の応用、拡張規格である
  • EthernetがLAN標準、802.11ファミリーは無線LAN(WLAN)と呼ばれる
  • 区別の理由
    • スケジューリングのメカニズムが違う
    • 「ランダムアクセスチャネル」として媒体を共有
    • パケットスケジューラが存在しない
    • すべてのデバイスは協調して共有チャネルのパフォーマンスを保証する動作をする必要がある
    • 衝突した際の回避方法が違う

衝突検知

  • Ethernet規格の衝突検知
    • CSMAプロトコル
      • 他の誰かが通信していないか確認
      • チャネルビジー状態 →　フリーになるまで聴き続ける
      • チャネルフリー状態 →　すぐにデータを送信する

衝突回避

  • 信号伝搬中に衝突が発生した場合の検知方法
    • Ethernet標準の衝突検知 → CSMA/CD
      • 衝突した双方がランダムに決定された時間だけ待機する(指数関数的バックオフ)
    • Wifiの衝突検知 → CSMA/CA
      • 信号伝搬中に衝突が検知できない場合の検知方法
      • 送信者がチャネルがアイドル状態を検知する
      • 検知後、メッセージ全体を一度に送信することで衝突回避する
      • WiFiフレーム送信後、次回送信前に受信者から明示的な受信確認を待つ

有効なチャネル利用率

  • 良好なチャネル利用率(衝突数の最小化)
    • チャネル使用率10%未満に抑える
    • 明示的な調節、スケジューリングなし
    • 利用率が高まると、衝突の数が増加しネットワーク全体が不安定になる

WiFi標準とその機能

  • IEEE 802委員会は新しいプロトコル排出し続けいている
  • 802.11b, gが広く使われ、サポートされている標準である
    • 周波数帯 : 2.4GHz ISMバンド(無免許帯)
    • チャネルあたりの帯域幅 : 20MHz
    • データストリームサポート : 最大1つ
    • 出力 : 最大200mW
  • 802.11n, ac
    • チャネルあたりの帯域幅 : 20MHz → 40MHz
    • 高位の変調方式採用
    • 複数ストリーム並行送信技術MIMO使用
    • (理想的状況下)acはギガビット/秒を越えるスループット実現できる

理想的な状況ってあるの？

  • WiFiネットワークは広く利用さている
    • パフォーマンス上、最大の困難発生
    • セル間干渉
    • セル内干渉
  • サービス品質ポリシーの設定はできる
    • 自身のネットワーク内のデータ転送速度等
    • しかし、他人からの干渉は設定できない
  • 2.4GHz帯はチャネルを3つ提供している
    • 1, 6, 11(, 13, 14)
    • 国・地域による
  • WiFiネットワークが近くに存在する場合、周波数帯の重複が起こる

WiFiのパフォーマンス

  • 帯域幅・レイテンシの保証、割り当てを行わない
  • 帯域幅は環境のSN比にもとづいて変化
  • 出力は200mWに制限される
  • 2.4GHz帯、5GHz帯と周波数帯が限られている
  • 周波数帯が重複するチャネルが割り当てられる
  • 複数人で同じ電波チャネルを使用することになる

WiFiのパケットロス

  • WiFi送信時に多数の衝突が発生する場合がある
    • TCPパケットロスが多数計測はされない
    • WiFiプロトコルのデータリンク層・物理層は独自の再送・エラー訂正メカニズムを持っている
    • ネットワークの上位層から衝突を隠蔽する
    • TCPパケットロスが有線ネットワークより高くなる場合はあまりない
    • 下位層での衝突回避・再送処理により、パケットの到達時間の変動が大きくなる

WiFiネットワーク最適化

  • WiFiのシンプルさと利便性から、最適化することでアプリケーションに役立てるかどうか検討してみる価値がある！

使い放題の帯域幅を利用する

  • WiFiは、
    • 使いたい放題のWAN
    • 許容量が高いデータキャップ
    • 高スループットを持つネットワーク
  • に接続されている
  • 大きいファイルのダウンロード、アップデート、ストリーミングは、WiFi上で行うべきである

変化する帯域幅に適応

  • 動画のストリーミング中に、帯域幅の割り当てが変動しても適応できるようにしておかないといけない
  • QoSポリシー・・・
    • ネットワークのパフォーマンスを予測可能にして、帯域幅を効果的に使用できる
    • 同じワイヤレスネットワークに接続するユーザ間のみにしか効かない
    • 重複しているWiFiネットワーク間には効かない
  • アダプティブビットストリーミング！！！！

変動するレイテンシに適応

  • レイテンシとは
    • デバイスに対してデータ転送などを要求してから、その結果が返送されるまでの遅延時間
  • 予測不可能
  • レイテンシに敏感なアプリケーションの場合、WiFi上での挙動について検討する必要がある
